Урок №13. Тестирование.


Что такое тестирование, с какими проблемами может столкнуться программист, избегающий тестирование своего кода, как правильно написать тест-метод - об этом и многом другом мы поговорим в 13 уроке.

Тест 1


Выберите неверные утверждения

  1. Программист не должен заниматься тестированием, для этого есть тестировщики
  2. Программист должен заниматься тестированием
  3. Правильно тестировать, выводя в методе мейн результаты на консоль
  4. Правильно тестировать, используя тест-классы, тест-методы с использованием библиотек jUnit/TestNG

Тест 2


Тестирование это -

  1. Поиск багов
  2. Попытка сломать программу
  3. Проверка соответствия реального поведения программы и ожидаемого
  4. Процесс запуска тестов

Задача 1


Написать тест-метод для задачи 1 из 12-го урока.

Задача 2


Написать тест-метод для задачи 2 из 12-го урока.

Ответы:

Тест 1

Увтерждения под номерами 1 и 3 не верны.

Тест 2

Правильный ответ 3.

Задача 1

@Test
public void whenFindMinElementInArrayThenResultThree() {
    int[] testArray = {5, 4, 3, 1, 2};
    int expected = 3;
    Array array = new Array();
    int result = array.min(testArray);
    assertThat(result, is (expected));
}

Задача 2

@Test
public void whenCalculateMediumArithmeticThenResultThree() {
    int[] testArray = {1, 2, 3, 4, 5};
    int expected = 3;
    Array array = new Array();
    int result = array.arithmetic(testArray);
    assertThat(result, is (expected));
}
